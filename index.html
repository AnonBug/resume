某测绘院互联网地图服务平台（实验室项目） | 项目 PM
  • Situation：一个实验室项目，甲方有许多历史地图成果需要向公众提供网络浏览服务，同时希望支持一定的专题定制。
  • Task：实验室有类似功能的历史项目，但技术栈较为落后，需要在理解历史代码的基础上进行重构，以支持项目要求。
  • Action：
    • 前端重构：将历史项目的 jQuery 技术栈切换至 Vuejs 生态
    • 地图框架：地图实现上，摒弃笨重的 ArcGIS Api for JavaScript ，选用轻量开源的 Leaflet 库及第三方插件实现。
    • 重写制图引擎：历史项目的定制部分，使用了后端渲染方案，前端展示后端返回的符号图片；借助前端图形库 D3js 、ECcharts 等开源符号库，在前端封装了一套 Leaflet 符号渲染系统，替换后端服务引擎
    • 启用 Node 服务端：原后端栈是 Java，由于语言的限制，加上抽离制图引擎后，后端服务相对简单，使用 Express 进行了重写
  • Result：
    • 开发效率提升：转至 Vuejs 技术栈后，借助其组件化、数据驱动的设计思想，提高了代码复用，降低了维护成本
    • 用户体验：引入 MVVM 框架使网页加载渲染速度更快；可交互的符号系统提升了用户体验
个人小项目
leaflet.charts 插件 | 仓库地址
  • 仓库介绍：一个向 Leaflet 地图插入点状分布式统计图的插件，支持 D3js 和 ECharts 图表。
  • 技术思路：
    1. 借助 Leaflet 提供的 L.divIcon 功能，向指定制图位置传入一批 div 容器
    2. 对容器进行批量管理，处理数据，生成指定统计符号
    3. 引入 ECharts 可以借助其动态图表的能力，快速生成动态可交互的统计图形
    4. 引入 D3js 能够支持更多异形符号
leaflet.snip 插件 | 仓库地址
  • 仓库介绍：一个框选截图的 Leaflet 插件，向地图工具栏中添加一个按钮，可在地图中画出一个矩形框并获取框选范围内的地图。
  • 技术思路：
    1. 通过监听鼠标事件，获取框选的起止点经纬度坐标
    2. 使用 Leaflet 提供的 api 将经纬度坐标转换为屏幕坐标
    3. 通过 html2canvas 库，获取地图容器截图
    4. 使用框选的屏幕坐标对容器截图进行截取，返回框选部分的截图
